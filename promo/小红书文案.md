## 正文长版

最近做了一个专门给中文系/人文学科用的 Agent Skill，名字叫《文科论文写作焚诀》。

它解决的不是“让 AI 帮我凑 8000 字”，而是文科论文里最难、也最容易翻车的几件事：

1. 先搜原文和研究论文，而不是上来就写。
2. 文献要进 Zotero，题录、PDF、可引用观点都要留痕。
3. 文献综述不是“甲说乙说丙说”，而是要梳理学术史、找分歧、找创新点。
4. 论文必须收束到一个中心问题，不能什么都想讲。
5. 小标题不能切得七零八落，每一节都要承担论证推进。
6. AI 最容易幻觉，所以每个“某某认为”都必须能和文献原文对上。
7. 写完后还要模拟严苛中文系教授审稿，再让作者回应、修订、核验引用、检查 Word 格式。

它更像一个论文工作室，而不是一个提示词：  
资料研究员、论文撰写者、审稿教授、作者回应者、修订者、引文核验员、格式审校员轮流上场，把论文从灵感生成变成可审计的学术生产流程。

目前已经在本地 Codex 里配置完成，支持 Zotero、Word、APA 文内引用、DOCX 格式体检和引文核验清单。后面准备整理成 GitHub 项目，让其他人的 Agent 也能直接学会这套写法。

一句话：  
AI 可以写论文，但必须被训练成一个知道证据、知道文献、知道结构、知道审稿标准的文科学术助手。

## 正文短版

做了一个中文系论文 Agent Skill：《文科论文写作焚诀》。

它不是普通提示词，而是一套完整论文工作流：检索原文 -> Zotero 入库 -> 文献综述 -> 提炼创新点 -> 搭结构 -> 写正文 -> C 刊式审稿 -> 作者回应 -> 深度修订 -> 引文逐条核验 -> Word 格式体检。

重点解决 AI 写文科论文最常见的问题：幻觉、假引用、文献综述空转、小标题太碎、中心论点散、格式不规范。

目标是让 Agent 不只是“会写”，而是按中文系论文的证据链和审稿逻辑写。
