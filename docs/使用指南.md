# 使用指南

## 一句话调用

```text
使用 $wenke-lunwen-xiezuo-fenjue，按 C 刊论文标准审查并重写这篇中文系论文，要求引用逐条核验，最后生成 Word。
```

## 推荐任务拆分

### 1. 资料阶段

让智能体先找原文、研究论文和理论文献，并保存到 Zotero。  
要求它说明每篇文献能支撑什么观点，不能支撑什么观点。

### 2. 综述阶段

要求智能体把文献组织成学术史问题，而不是罗列摘要。  
综述应当产生创新点，而不是作为正文前的装饰。

### 3. 写作阶段

要求智能体围绕一个中心问题写作。  
如果文章开始谈太多方向，应立刻要求它压缩主轴。

### 4. 审稿阶段

让智能体扮演严苛中文系教授，优先批评：

- 中心论点散。
- 结构碎。
- 文献引用不真实。
- 文本细读不足。
- 格式不符合要求。

### 5. 修订阶段

让智能体扮演有自我判断的作者，不机械服从审稿意见，而是形成修订方案，再重写正文。

### 6. 引文核验阶段

逐条检查正文中的文内引用。  
所有“某某认为”都必须能和 Zotero 文献、PDF 摘录或 evidence 文件对应。

## 最佳实践

- 一开始就告诉智能体引用格式，例如 APA、GB/T 7714、脚注制。
- 不要只要求“写得学术一点”，要要求“观点和文献逐条对应”。
- 对中文系论文，原文细读永远是核心，不要让概念堆叠压过文本。
- 如果论文被切成太多小标题，要求合并大节，并说明每节的推进功能。
- 如果生成 Word，必须做格式体检。
