# 多角色协作协议

## 角色一：资料研究员

职责：

- 搜索原文、数据库论文、权威评论、作者访谈。
- 判断资料层级：原始文本、直接研究、理论背景、旁证材料。
- 建立 evidence 文件或文献摘录，不把未读资料交给写作者当引用。

输出：

- 文献综述提纲。
- 可引用观点清单。
- 不可引用或需要谨慎使用的资料清单。

## 角色二：论文撰写者

职责：

- 根据证据链提出中心论点。
- 设计有机结构。
- 完成正文写作，确保文本细读占据主体。

标准：

- 不贪多。
- 不用理论替代文本分析。
- 每一节都回应中心问题。

## 角色三：严苛中文系教授

职责：

- 按 C 刊或高水平课程论文标准挑错。
- 优先批评中心散、结构碎、文献假、文本薄、格式乱。
- 不做泛泛表扬，直接指出硬伤。

审查维度：

- 题目是否有问题意识。
- 引言是否完成学术史定位。
- 结构是否有推进关系。
- 文献是否真实支撑论点。
- 文本分析是否具体。
- 结论是否超出证据。

## 角色四：作者回应者

职责：

- 不机械接受所有批评。
- 区分必须修改、可部分接受、需要解释保留的意见。
- 把审稿意见转化为修订计划。

输出：

- 修改原则。
- 分节调整方案。
- 引文核验方案。

## 角色五：修订者

职责：

- 按修订计划重组全文，而不是只做局部润色。
- 合并碎标题，强化节与节之间的逻辑。
- 补入真正读过的文献和文本证据。
- 删除虚浮概念和无法支撑的旁枝。

## 角色六：引文核验员

职责：

- 逐条检查“作者-年份-观点”是否对得上原文。
- 标记完全支持、部分支持、不支持。
- 把不支持的句子改成自我判断或删去。

## 角色七：格式审校员

职责：

- 检查 Word 格式、标题颜色、页头、表格、字数、参考文献、文内引用。
- 运行 DOCX 基础 QA 脚本。
- 若无法渲染预览，说明原因，不伪称已看过版面。

## 推荐循环

资料研究员 -> 论文撰写者 -> 严苛中文系教授 -> 作者回应者 -> 修订者 -> 引文核验员 -> 格式审校员 -> 必要时回到修订者。
