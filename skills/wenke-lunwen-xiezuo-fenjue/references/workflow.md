# 文科论文证据链写作流程

## 0. 任务建模

先把用户需求转化为论文工程参数：

- 研究对象：作品、作家、概念、时期、文体、理论问题。
- 文体目标：课程论文、C 刊模拟、投稿论文、开题报告、文献综述、修改稿。
- 评价标准：字数、格式、引用体系、是否必须使用 CNKI、是否需要 DOCX。
- 约束条件：不得用图表、不得碎标题、必须有文内引用、必须核验文献原文。

如果用户没有给出题目，先根据材料建立“问题句”，例如：“本文讨论某作品如何通过某种互文机制，把古典经典转化为现代小说的叙事结构。”

## 1. 材料收集

材料分三层，顺序不能颠倒：

1. 原始文本：作品原文、古典文本、权威版本、作者自述。
2. 直接研究：围绕作品、作者、关键词的论文。
3. 理论与谱系：互文性、改写、戏仿、经典化、文学史、文体学等更大的研究。

保存到 Zotero 时，为每条资料保留：

- 标题、作者、刊物、年份、DOI 或 URL。
- PDF 或可检索文件。
- collection 名称。
- 是否已读、是否可引用、可引用段落。

## 2. 文献综述不是资料堆放

文献综述要回答四个问题：

1. 已有研究围绕哪些关键词展开。
2. 学界已有共识是什么。
3. 分歧和空白在哪里。
4. 本文如何从文本细读进入这个空白。

不要把综述写成“甲说、乙说、丙说”的流水账。每一组文献都要被组织成一个学术问题，例如“互文性研究从影响关系转向文本机制”“当代小说研究从主题阐释转向古典资源的结构化再生产”。

## 3. 中心论点压缩

好的文科论文只抓一个中心问题。检验方法：

- 能否用一句话说清本文要证明什么。
- 每节标题是否都能回到这句话。
- 每条文献是否服务于这个问题。
- 每段文本细读是否在推进论证，而不是展示知识。

中心论点建议使用这种句式：

“本文认为，某作品并非简单借用某经典意象，而是把经典中的某种结构转化为现代小说的某种叙事机制；这种机制使作品在纵向文学史谱系与横向作家创作中获得特殊位置。”

## 4. 结构设计

文科论文的小标题不是目录装饰，而是论证路线。常用结构：

1. 引言：提出问题、梳理研究、限定中心、说明方法。
2. 第一节：把研究对象放进纵向谱系，说明为什么值得讨论。
3. 第二节：进入作品内部，分析核心文本机制。
4. 第三节：横向比较同作者、同类型或同传统作品，凸显个案意义。
5. 第四节：回到理论层面，说明该个案对现代文学与古代文学互文关系的启发。
6. 结语：收束创新点，不重复正文。

如果用户批评“小标题太碎”，应合并为 3 到 5 个大节，让每节内部容纳纵深展开。

## 5. 写作原则

- 引言要尽早点明文学史谱系，不要等到结尾才补。
- 理论术语只在能解释文本时出现。
- 文本分析要有“引文或情节事实-解释-回到论点”的链条。
- 比较研究要有比较项，不能只是并列提及。
- 结尾不另起新论题，只提升本文已经证明的内容。

## 6. 交付清单

完成前至少检查：

- 字数是否达到要求。
- 正文是否有文内引用。
- 参考文献是否和正文引用一致。
- 所有“某某认为”是否能追溯到已读文献。
- 是否误用图表、页头、颜色、脚注或尾注。
- DOCX 是否可打开，必要时是否渲染预览。
