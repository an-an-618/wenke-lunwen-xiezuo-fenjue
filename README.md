# 文科论文写作焚诀

面向中文系与人文学科论文写作的智能体 Skill。

它不是一个“帮你把题目扩写成八千字”的提示词，而是一套把论文写作拆解为证据链工程的工作流：检索原文、整理 Zotero、阅读文献、生成综述、提炼问题意识、搭建有机结构、完成正文、进行严苛审稿、逐条核验引用，最后交付符合格式要求的 Word 文档。

## 适用场景

- 中文系、古代文学、现当代文学、比较文学、文艺理论方向课程论文。
- 需要 CNKI、Zotero、Word、APA 或其他引用格式配合的写作任务。
- 需要从原典和研究文献中提炼观点，而不是让 AI 凭空发挥。
- 需要模拟 C 刊审稿、专家批评、作者回应与深度修订。
- 需要对“引用是否真实”“文献观点是否对得上原文”进行专项审校。

## 核心能力

- **证据优先**：先读原文和文献，再下判断。
- **Zotero 入库**：围绕选题建立 collection，保存题录、PDF 和可引用摘录。
- **文献综述**：把研究史梳理成问题谱系，而不是资料堆砌。
- **中心论点压缩**：避免“什么都想说”，把论文收束到一个可证明的问题。
- **结构重组**：减少碎标题，让每一节承担论证推进功能。
- **多角色协作**：资料研究员、论文撰写者、严苛教授、作者回应者、修订者、引文核验员、格式审校员轮流工作。
- **引文反幻觉**：所有“某某认为”都必须能和原文证据对应。
- **Word 体检**：检查字数、表格、页头、标题颜色、引用格式等常见硬伤。

## 工作流总览

1. 明确论文任务、学科规范和交付格式。
2. 搜集原典、作品原文、权威研究和理论文献。
3. 将文献保存到 Zotero，并标记“已读”“可引用”“待核验”。
4. 从文献综述中提取学术史、关键词、分歧和创新点。
5. 形成一个中心论点，避免主题散乱。
6. 设计有机小标题结构，减少机械切分。
7. 写作正文，保证原文细读、二手文献和自我判断边界清楚。
8. 进入多角色审修循环。
9. 对文内引用逐条核验。
10. 生成 Word 文档并进行格式体检。

## 设计理念

AI 写文科论文的真正难点，不是“写得像论文”，而是让每一个判断都能回到材料、文献和论证结构中。  
《文科论文写作焚诀》把论文生产拆成可审计的流程，让智能体在写作中不断追问：

- 这个观点从哪里来？
- 这个文献真的这么说了吗？
- 这段文本细读是否证明了中心论点？
- 小标题之间是否存在推进关系？
- 最终 Word 文档是否符合学术格式？
